Git

Introduction to Git

Git was created in 2005 by Linus Torvalds, the founder of Linux, as a way to keep track of
files on a computer. Git is essential in the world of Web Development, Programming, and Computer
science.

Git works by uploading files to a remote repository, often Github, in series of updates called
Commits. Only certain computers are allowed to upload to the repository, but anybody can download
the files hosted. Github is often used by small developers/community devs to host projects.
Github can also host a basic website. In this class, Git will be used to upload and check work.

Git Commands

Git is acessed via the unix command line, and is used via the unix command line. There are
many different commands within git that can be used, but for this class, you will only need these
Basic commands.

git status
Gets current status of staged files, local repo, and the remote.

git add
Stages the specified file (after command) for commit

git commit -m "message"
Commits changes made to staged files to local repository. "Message" may be changed to the
message to appear on the commit.

git push
pushes changes in local repo to the remote

You may use other git commands in your time in this course. That's fine, but these 4 basic
commands are the prominent ones. Whenever you make changes to your repo, all four will be used
try to memorize these, it will do you good to make a habit of commiting to you repo after you are
done making changes.

Git Stages

Git has three stages where files can be stored. On the computer itself, the local repository,
and the remote repository. The remote repository is the only place where files are stored not on
the computer itself. First, files are staged for commit with various git commands, such asgit add. They are commited to the local repository with git commit -m "message". Finally, the changes are made to the remote with git push.

Git Setup

Setting up your repository is an involved process, and takes a long time. Chris or Jeff will
help you, but this is the basic process.

Step 1:

Create a Github account using a personal email if you do not already have one. Next, create a
new repository. if you have not already, run the git command from the terminal to download git

Step 2:

Run the ssh keygen -t rsa command from your terminal. Skip the questions by pressing enter
A new folder should be created in ~/Users/yourusername. it should be called .ssh. Inside, there should Be two files, one named id_rsa
and another named id_rsa.pub. Your filenames may vary. but the .pub should still exist.

IMPORTANT!

The file named id_rsa is your private key. NEVER under ANY CIRCUMSTANCES
should it go onto the internet. This key allows encrypted data going to your computer to be
decrypted. Which poses a MASSIVE security risk.

Step 3:

View your public key by running the command cat ~/.ssh/id_rsa.pub. Copy the entire text
starting from ssh-rsa. Click your profile picture in the github homepaage, navigate to "settings",
SSH and GPG Keys, click "New SSH Key"

Step 4:

Paste the text output from the terminal into the window. Set the title as "My school MacBook" or
another appropriate name. When you are done, click "Add SSH Key"

Step 5:

Click "Create Repository" in the homepage. For the title, your repository name needs to be
{your github username}.github.io. This is case-sensitive. Change no other settings and click
"Create Repository". Once you are done, a repository setup page will appear, do not go further
but leave the page open and return to the terminal.

Step 6:

Using the terminal, run the following commands. This will create a folder on your desktop where
you can store your website code.

cd Desktop
mkdir website
cd website

Step 7:

Run these two commands. replace username and email with your github username and the email you used
to setup github. 

git config --global user.name "username"
git config --global user.email"email"
ensure that you are in the website directory by using pwd

Step 8:

On github, select the SSH button. You will see a series of commands in the window below. Copy
these commands using Ctrl-C or the icon. Run these commands in your terminal, and type "yes" to
any questions that pop up. Once you are done, refresh your github page, and it should have changed.

After this step, your setup is complete. The repository and your website folder are linked.
There should be some various files in the website folder. ignore these for now until you are
told to edit them.
